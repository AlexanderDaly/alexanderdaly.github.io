# Security Policy

## Supported Versions

This repository powers the public GitHub Pages website for `alexanderdaly.github.io`.

| Version / Branch | Supported |
| --- | --- |
| `main` / current live site | ✅ Supported |
| Pull requests awaiting review | ✅ Reviewed before merge |
| Old commits, forks, or archived copies | ❌ Not supported |

## Reporting a Vulnerability

If you discover a security issue affecting this website or repository, please report it responsibly.

Preferred reporting path:

1. Use GitHub’s **Report a vulnerability** / private vulnerability reporting flow for this repository, if available.
2. If private reporting is not available, open a GitHub issue with a brief, non-sensitive message asking for a secure contact path. Please do **not** include exploit details, secrets, payloads, or proof-of-concept code in a public issue.

GitHub notes that private vulnerability reporting is separate from `SECURITY.md` and only works for public repositories where the feature is enabled. If it is not enabled, reporters should follow the repository security policy or ask maintainers for a preferred security contact.

## What to Include

Please include as much of the following as you safely can:

- Affected page, URL, branch, or file.
- A clear description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact.
- Any suggested fix or mitigation.
- Whether the issue appears to expose private data, secrets, credentials, or deployment configuration.

## Scope

Security reports are especially useful for:

- Cross-site scripting or unsafe HTML/script behavior.
- Exposed secrets, tokens, credentials, or private configuration.
- GitHub Pages deployment or repository permission risks.
- Vulnerable dependencies, build tools, or third-party scripts.
- Misconfigurations that could affect visitors or repository integrity.

The following are generally out of scope unless they demonstrate a real security impact:

- Generic best-practice suggestions without a vulnerability.
- Social engineering.
- Denial-of-service testing.
- Automated scanner output without reproduction details.
- Issues affecting only unsupported forks or old commits.

## Response Expectations

I will make a best effort to:

- Acknowledge the report as soon as practical.
- Investigate and validate the issue.
- Prioritize fixes based on severity and exploitability.
- Coordinate disclosure when appropriate.
- Credit the reporter if they want credit and the report is valid.

Please avoid public disclosure until the issue has been reviewed and, where needed, fixed.

## Safe Harbor

I welcome good-faith security research. I ask that you:

- Do not access, modify, or delete data that is not yours.
- Do not disrupt the website or GitHub repository.
- Do not use social engineering, phishing, or physical attacks.
- Only test against systems you are authorized to test.
- Report findings promptly and privately.

Good-faith reports that follow this policy are appreciated.
